Phase 1 — Foundations & Exploration
Week 1–2
👤 Dev A — AI Concepts & Theory
  • What is Machine Learning? Supervised vs unsupervised vs reinforcement learning
  • What is NLP (Natural Language Processing)? Use cases and examples
  • Difference: LLM vs SLM vs RAG — when to use which approach
  • Read: "The Illustrated Transformer" by Jay Alammar (free online)
  • Study: Tokenization, embeddings, and vector representations
  • Set up: Python 3.10, pip, venv, Jupyter Notebook on your laptop
  • Run: A basic text classification demo using HuggingFace Transformers
  • Explore: HuggingFace model hub — browse medical NLP models
👤 Dev B — Audio & Speech Basics
  • Study: How does speech-to-text work? Acoustic model + language model explained
  • Learn: Whisper architecture by OpenAI — what makes it multilingual?
  • Set up: PyAudio / sounddevice library for microphone input on laptop
  • Test: noisereduce library — apply noise cancellation to a sample recording
  • Build: A simple Python script — press Enter to record, press Enter to stop, saves .wav file
  • Explore: Whisper for transcription of English audio (run locally)
  • Research: How Whisper handles multilingual and code-switched audio
  • Test: Record yourself speaking Gujarati and run through Whisper base model
🤝 Joint Activities (Do Together)
Daily 15-min standup call Watch: 3Blue1Brown Neural Networks series (YouTube) Set up shared GitHub repo for the project Read HuggingFace intro docs together Friday: Teach each other what you learned this week (30 min)
🎯 Week 1 Deliverable
🎯
Demo to each other on Friday: Dev B shows a working Python script that records audio from mic, saves as .wav file, and transcribes it with Whisper. Dev A runs a basic NLP text classification model and explains tokenization. Both explain 5 key AI terms to each other in their own words.
👤 Dev A — SLM Study & Setup
  • What is a Small Language Model (SLM)? How is it different from GPT-4?
  • Study: DistilBERT, TinyLlama, Phi-2 — architecture overview (read papers or blog summaries)
  • Understand: Training vs fine-tuning vs inference — what happens at each stage?
  • Learn: LoRA (Low-Rank Adaptation) — how to fine-tune a model without a massive GPU
  • Set up: HuggingFace Transformers + PEFT library (for LoRA) locally
  • Run: TinyLlama inference locally — generate text from a prompt
  • Study: What data format do you need to fine-tune a model? (instruction format, JSON)
  • Prepare: A small dataset of 50 doctor-patient dialogue examples (can be in English for now)
👤 Dev B — Gujarati Whisper Fine-tuning
  • Research: Mozilla Common Voice dataset — download the Gujarati audio samples
  • Prepare: Convert and clean dataset into the format Whisper expects
  • Fine-tune: Whisper-small on Gujarati using HuggingFace Trainer API
  • Evaluate: Calculate WER (Word Error Rate) on held-out test audio
  • Test: Record your own Gujarati speech and run through fine-tuned model
  • Handle: Mixed Gujarati + English (code-switching) — common in doctor conversations
  • Optimize: Reduce file size / speed up inference on CPU
  • Save: Fine-tuned model checkpoint and share on GitHub repo
🤝 Joint Activities
Discuss: SLM vs RAG — which fits our use case better? Watch: Andrej Karpathy "Let's build GPT" (YouTube) Friday knowledge share: each explains the week's key concept in plain Gujarati/English Write a shared doc: "What we understand about AI so far"
🎯 Week 2 Deliverable
🎯
Demo to each other on Friday: Dev B shows a fine-tuned Whisper model that correctly transcribes Gujarati sentences into text. Dev A can run TinyLlama inference locally, explain what fine-tuning means, and show a sample training JSON dataset.
Phase 2 — Build the Product
Week 3–4
👤 Dev A — RAG Pipeline
  • Study: What is RAG (Retrieval-Augmented Generation)? Concept explained simply
  • Learn: Vector databases — set up ChromaDB or FAISS locally (both free, no cloud needed)
  • Understand: Sentence embeddings — use sentence-transformers library to convert text → vectors
  • Build: Ingest medical terminology PDFs / docs into ChromaDB as searchable knowledge base
  • Build: Query pipeline — input text → similarity search → retrieve top-3 relevant chunks
  • Combine: RAG + SLM — feed retrieved context into TinyLlama prompt to improve accuracy
  • Test: Ask medical questions ("What is hypertension?") and verify correct retrieval
  • Document: Draw a diagram of the full RAG pipeline for the presentation
👤 Dev B — SOAP Note Generator
  • Study deeply: What is a SOAP note? Subjective / Objective / Assessment / Plan — with examples
  • Collect: 20–30 sample doctor–patient conversation transcripts (English, from public datasets)
  • Fine-tune: TinyLlama or Phi-2 on transcript → SOAP pairs using LoRA
  • Build: Prompt engineering — design prompt template that reliably outputs structured SOAP
  • Build: Summary generator — extract key symptoms + diagnosis from transcript in 3–5 lines
  • Test: Input a mock Gujarati–English transcript (translated) → verify SOAP output in English
  • Evaluate: Check that all 4 SOAP sections are correctly and fully populated in every test
  • Refine: Improve fine-tuning dataset based on bad outputs you observe
🤝 Joint Activities
Integrate: Connect Whisper output → SOAP model input (first end-to-end test) Discuss: When to use RAG vs fine-tuning — write down your conclusions Write shared technical architecture document (for Week 5 presentation) Friday demo: Show full text pipeline — transcript in, SOAP out
🎯 Week 3 Deliverable
🎯
End-to-end text pipeline working: Input a text transcript → RAG retrieves relevant medical context → SLM generates a SOAP note + 5-line summary in English. Both developers can explain how each component works to each other clearly.
👤 Dev A — Integration & Pipeline
  • Build: End-to-end pipeline class — audio recording → Whisper → RAG → SLM → SOAP output
  • Handle: Long sessions (2 to 60 minutes) by chunking audio in 30-second segments
  • Add: Real-time transcription display — stream transcript chunks as doctor speaks
  • Handle: Speaker diarization — try to label "Doctor:" vs "Patient:" turns
  • Error handling: Silence detection, microphone failure, out-of-memory errors
  • Output: Save transcript + SOAP + summary to .txt and .pdf file automatically
  • Performance test: Full 10-minute recording → measure time to generate SOAP output
  • Optimize: Where are the bottlenecks? Can you speed up any step?
👤 Dev B — Desktop UI (Python)
  • Choose: Tkinter (simpler) or PyQt5 (nicer) for Python desktop application
  • Build: Main window with large, clear START (green) / STOP (red) buttons
  • Build: Live transcript viewer — scrollable text box that updates in real time
  • Build: Status bar — shows "Recording... 00:04:32 elapsed" while active
  • Build: Output panel with 3 tabs — Transcript | Summary | SOAP Note
  • Add: Doctor name + Patient name input fields at top of screen
  • Add: Export button — saves output as .txt file, with copy-to-clipboard option
  • Polish: Clean layout, readable fonts, works on 1080p laptop screen
🤝 Joint Activities
User test with a real doctor or friend (if possible) Fix the top 5 bugs found in testing Measure accuracy: how correct are the SOAP notes? Write README + step-by-step setup guide for the app Record a 2-min screen recording of the app working as backup for demo day
🎯 Week 4 Deliverable
🎯
Fully working desktop application: Doctor opens the app on a laptop, types their name, presses START, speaks Gujarati for 5+ minutes, presses STOP — and sees a full English Transcript + 5-line Summary + SOAP Note on screen. Can export to a file.

Python Libraries to Install
  • transformers — HuggingFace (for SLM inference + fine-tuning)
  • peft — LoRA fine-tuning adapter (lightweight training)
  • openai-whisper — speech recognition (run locally)
  • chromadb — local vector database for RAG
  • sentence-transformers — generate text embeddings
  • sounddevice + noisereduce — audio recording & cleanup
  • PyQt5 or Tkinter — desktop UI for the app
  • reportlab — generate PDF output files
Datasets to Use
  • Mozilla Common Voice — Gujarati audio dataset (free)
  • MTS-Dialog — medical dialogue + SOAP note pairs
  • ACI-Bench — clinical conversation benchmark dataset
  • WikiMed — medical Wikipedia articles for RAG knowledge base
  • Record your own Gujarati samples for testing
Recommended Models (Free & Local)
  • openai/whisper-small — Gujarati ASR base model
  • TinyLlama-1.1B-Chat — fast SLM for SOAP generation
  • microsoft/phi-2 — stronger SLM, slightly heavier
  • sentence-transformers/all-MiniLM-L6-v2 — embeddings for RAG
  • All models downloadable free from huggingface.co
